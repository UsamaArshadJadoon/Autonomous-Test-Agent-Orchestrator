⚡ Quick Start (5 minutes)

Step 1: Installation

cd test-agent-orchestrator npm install

Step 2: Configure Credentials

cp .env.example .env # Edit .env with your Jira token, database URL, API keys

Step 3: Setup Database

psql -U postgres -f db/schema.sql

Step 4: Build Project

npm run build

Step 5: Run Your First Test

test-agent run SEK-1934 --env qa --mode full
✓ That's it! The system will automatically:
  • Fetch your story from Jira
  • Generate comprehensive test cases
  • Execute tests in your QA environment
  • Log bugs back to Jira
  • Send results to Slack/CI

🏗️ System Overview

ATAO is a 7-agent orchestration system that automates the entire testing workflow. Here's how it works:

┌─────────────────────────────────────────────────────────────┐ │ Your Command: test-agent run SEK-1934 --env qa --mode full │ └────────────────┬────────────────────────────────────────────┘ │ ┌───────┴───────┐ │ 7 Agents │ └───────┬───────┘ │ ┌────────────┴─────────────┐ │ │ ┌───▼──┐ ┌──────┐ ┌──────┐ │ │Story │→ │Test │→ │Gap │→ │ │Agent │ │Writer│ │Check │→ │ └──────┘ └──────┘ └──────┘ │ │ ┌────▼────┐ │Executor │ └────┬────┘ │ ┌──────────┤ │ │ ┌───▼───┐ ┌───▼──┐ │Review │→ │Approve └───┬───┘ └────┬─ │ │ ┌───▼──────────▼──┐ │ Bug Logger │ │ + Webhooks │ └─────────────────┘

The 7 Agents

Agent Role Input Output
Story Agent Reads from Jira Story key (SEK-1934) Story details + AC
Test Writer Generates tests Story details 45-60 test cases
Gap Analyzer Validates coverage Tests + AC Coverage report
Test Executor Runs tests (Playwright) Test cases Pass/fail + screenshots
Reviewer QA review gate Results Approval/rejection
Bug Logger Creates Jira issues Failed tests Linked bug tickets
Orchestrator Coordinates all Initial request Final report

🔧 Troubleshooting

Error: Story Not Found

Problem

Error: STORY_NOT_FOUND Story SEK-1934 not found in Jira

Solutions

  1. Verify the story key is correct: https://jira.company.com/browse/SEK-1934
  2. Check your Jira token in .env is valid
  3. Ensure you have permission to view the story
  4. Test connection: curl -H "Authorization: Bearer $JIRA_TOKEN" https://jira.company.com/rest/api/3/issues/SEK-1934

Error: Database Connection Failed

Problem

Error: DATABASE_UNREACHABLE Database is unreachable. Check DATABASE_URL and credentials.

Solutions

  1. Verify PostgreSQL is running: psql -U postgres -c "SELECT 1"
  2. Check .env DATABASE_URL is correct
  3. Ensure schema is loaded: psql -U postgres -f db/schema.sql -d app_qa
  4. Check credentials: psql postgresql://user:pass@localhost:5432/app_qa

Error: App Unreachable

Problem

Error: APP_UNREACHABLE App is unreachable at https://qa.app.company.com

Solutions

  1. Verify URL in .env: cat .env | grep APP_URL
  2. Test connectivity: curl https://qa.app.company.com -I
  3. Check firewall/VPN is connected
  4. Verify environment is up with your DevOps team

Tests Are Flaky/Intermittent

Problem

Tests pass sometimes, fail others. Status shows "FLAKY".

Solutions

  1. Check app performance: curl https://qa.app.company.com -w "Time: %{time_total}s"
  2. View flaky test registry: test-agent flaky-registry SEK-1934
  3. Run in regression mode to track baseline: test-agent run SEK-1934 --mode regression
  4. Check if app is under heavy load

API Rate Limit Exceeded

Problem

Warning: CLAUDE_API_RATE_LIMIT Anthropic API rate limit exceeded. Waiting before retry...

Solutions

  1. System automatically retries with exponential backoff - wait 30 minutes
  2. Check quota: https://console.anthropic.com/account/billing
  3. Contact Anthropic support for higher limits
  4. Stagger test runs: Wait 30 min between different stories
Need Help? Contact qa-team@company.com or post in #qa-testing Slack channel
